## ステップ1: 朝の宣言の振り返り

- `daily/YYYY-MM-DD.md` を読み、朝にやると言ったことを1つずつ確認する
- やれた場合: 具体的に何ができたかを記録。成果物があるなら確認する
- やれなかった場合:
  - 「なぜやれなかった？」→「それはなぜ？」→ 根本原因まで掘る
  - 環境や考え方をどう変えればよいかを一緒に考える
  - 発見を USER.md に反映���る

## ステップ2: kanban.md の完全性チェック

全タスクについて以下を確認し、不足があればその場で埋める:

- [ ] ゴール状態が書かれているか？曖昧でないか？
- [ ] 現在の状態が最新か？
- [ ] ネクストアクションが書かれているか？1時間以内で完了できる粒度か？
- [ ] ネクストアクションの「最初の10秒」が言語化されているか？
- [ ] 長期間ネクストアクションが変わっていないタスクはないか？（停滞の兆候）
- [ ] 締め切りが設定されていないタスクはないか？
- [ ] 保留状態で放置されているタスクはないか？（「いつかやる」リストは禁止）

**情報が不足している限り、セッションを終わらせない。** その場で一緒に考えて記入する。

## ステップ2.5: errands.md の棚卸し

errands.md の未完了雑務を全件チェックする。

- 完了したものは `[x]` に変更済みか確認
- 追加日から3日以上経過しているものをリストアップ
  - 3-4日: 「[X]が[N]日目。明日こそやろう。最初の10秒は？」
  - 5日以上: 判断を強制。「(1)今やる (2)明日の具体的な時間を決める (3)不要なら消す」
  - 「いつかやる」「そのうちやる」は受け付けない
- 新しく溜まった雑務がないか: 「今日やるべきだったのに追加し忘れた雑務ない？」
- 全部完了していたら: 「雑務ゼロ、クリーンな状態。いいね」

## ステップ3: 改善の言語化

- 仕事や人生面で「もっとこうやれた」ということを1つ言語化させる
- 5分で終わるインパクトのある打ち手がないか考えさせる
- 労力が大してかからないのにやっていないことがあれば指摘��る

## ステップ3.5: USER.md 更新

今日の対話を振り返り、以下に該当する発見があれば USER.md に追記する。**該当なしならスキップして良い。毎日必ず書く必要はない。**

チェックリスト:
- [ ] **有効だった手法**: 今日うまくいったアプローチはあるか？（例: 「5分だけと言ったら実際に30分集中できた」「雑務を先にやらせたらエンジンがかかった」）→ 「有効だったマネジメント手法」に実例として追記
- [ ] **逆効果だった手法**: 今日裏目に出たアプローチはあるか？（例: 「締め切りプレッシャーをかけたらフリーズした」「朝一で重いタスクを入れたら回避が発動」）→ 「逆効果だったマネジメント手法」に実例として追記
- [ ] **エネルギーパターン**: 新しい傾向が見えたか？（例: 「MTG後は必ずエネルギーが落ちる」「水曜は慢性的に低い」）→ 「エネルギーパターン」に追記
- [ ] **回避パターン**: 新しい回避の実例があったか？（例: 「[X]の前に毎回体調シールドが出る」「計画の作り直しで2時間消えた」）→ 「回避パターン実例」に追記
- [ ] **特性の発見**: ユーザーの働き方について新しく分かったことがあるか？→ 「特性・傾向」に追記

書く時のルール:
- 具体的な実例ベースで書く。「プレッシャーが効いた」ではなく「"15分で終わらせよう、15分後に聞く"と言ったら実際に12分で完了した」
- 日付を添える（いつの発見か後で分かるように）

## ステップ4: 翌日の準備

- 翌日やるべきことの優先度を確認
- 持ち越しタスクの扱いを決定させる:
  - やる → kanban.md のネクストアクションを更新
  - 取り下げる → kanban.md から削除。明確な判断として記録
  - 「なんとなく後で」は許さない

## ステップ5: 記録

`daily/YYYY-MM-DD.md` の実績と振り返りセクションを記入:

```markdown
## 実績
- タスク1: [完了/未完了] — [具体的な成果]
- タスク2: ...
- タスク3: ...

## 振り返り
- うまくいったこと: ...
- うまくいかなかったこと: ...
- 根本原因: ...
- 明日への改善: ...
```

## ステップ6: 翌日の朝チェックインが未スケジュールの場合

翌朝のセッション（cron タスク）が存在していることを確認する。なければ登録する。
